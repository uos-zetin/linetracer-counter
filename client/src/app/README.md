# `app/` 디렉터리

최상위 **엔트리(entry) 레이어**로서, 브라우저에 첫 번째 로드되는 코드와 전역 환경을 모두 여기서 초기화합니다.
`app`을 제외한 다른 레이어는 이 폴더를 **직접 import할 일이 없습니다.**

---

## 1. 역할 · 책임

| 구분                   | 설명                                     |
| ---------------------- | ---------------------------------------- |
| **엔트리 포인트**      | React 앱 초기화 및 DOM 마운트            |
| **전역 Provider 스택** | 라우터, 테마 등 앱 전역 컨텍스트 관리    |
| **라우팅**             | URL과 페이지 컴포넌트 매핑 (lazy import) |
| **전역 스타일**        | Tailwind 설정 및 디자인 토큰 정의        |

---

## 2. 구조 · 파일 설명

```
app/
├─ index.tsx
├─ index.css
├─ providers/
│  ├─ app-providers.tsx    # 모든 Provider를 조합
│  ├─ [provider-name].tsx  # 각 Provider 구현
│  └─ index.ts
└─ routing/
   ├─ routes.tsx
   └─ index.ts
```

| 파일/폴더        | 역할 및 제약사항                                          |
| ---------------- | --------------------------------------------------------- |
| **`index.tsx`**  | 앱 부트스트랩 전용. 다른 레이어에서 import 금지           |
| **`providers/`** | 전역 Provider 관리. kebab-case 명명, `children` 패턴 준수 |
| **`routing/`**   | 페이지 라우팅 정의. 데이터 로직 금지, lazy import 사용    |
| **`index.css`**  | 전역 스타일 및 디자인 토큰. `@theme` 영역에 UOS 토큰 정의 |

---

## 3. import 규칙

- **허용**: `pages/*`, `widgets/*`, `features/*`, `entities/*`, `shared/*`
- **금지**: 어느 레이어에서도 `app/*`을 import하는 행위
  → `app`은 **루트전용**. 다른 레이어가 필요하면 하위 레이어로 옮겨야 함.

> 이 README를 업데이트할 때는 **역할 · 구조 · 규칙** 3가지만 명확히 유지해 주세요. 후임 개발자가 “app은 부트스트랩 전용 레이어”임을 바로 이해할 수 있도록 하는 것이 목적입니다.
