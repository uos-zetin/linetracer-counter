# `shared/` 디렉터리

도메인 독립적이고 **모든 레이어에서 재사용 가능한 코드**를 제공하는 레이어입니다.
UI 컴포넌트, 유틸리티, 설정, 공통 타입 등 **프로젝트 전반에서 사용되는 기초 요소**를 구현합니다.

---

## 1. 역할 · 책임

| 구분              | 설명                                     |
| ----------------- | ---------------------------------------- |
| **UI 프리미티브** | 기본 UI 컴포넌트 (버튼, 입력, 아이콘 등) |
| **유틸리티**      | 순수 함수, 헬퍼 함수                     |
| **설정**          | 전역 설정, 상수, 환경 변수               |
| **공통 타입**     | 프로젝트 전반에서 사용하는 타입 정의     |
| **API 기반**      | HTTP 클라이언트, 공통 API 유틸리티       |

---

## 2. 구조 · 파일 설명

```
shared/
├─ ui/                # UI 프리미티브 컴포넌트
├─ lib/               # 유틸리티 함수, 공통 타입
├─ api/               # API 기반 설정
├─ config/            # 전역 설정, 상수
└─ index.ts           # Public API (필수)
```

| 파일/폴더      | 역할 및 제약사항                                   |
| -------------- | -------------------------------------------------- |
| **폴더명**     | kebab-case 사용                                    |
| **`ui/`**      | 도메인 독립적인 기본 UI 컴포넌트                   |
| **`lib/`**     | 순수 함수, 유틸리티, 공통 타입                     |
| **`api/`**     | HTTP 클라이언트, 공통 API 설정                     |
| **`config/`**  | 전역 설정, 상수, 환경 변수                         |
| **`index.ts`** | **Public API 정의. 외부에서는 이것만 import 허용** |

---

## 3. import 규칙

- **허용**: `shared/*` (내부)
- **금지**: `app/*`, `pages/*`, `widgets/*`, `features/*`, `entities/*`

---

## 4. 설계 원칙

### 도메인 독립성

- 비즈니스 로직, 도메인 상태 포함 금지
- 어떤 프로젝트에서도 재사용 가능한 수준 유지

### 순수성

- 사이드 이펙트 최소화
- 순수 함수 중심 설계
- 외부 의존성 최소화
