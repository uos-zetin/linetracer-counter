# `widgets/` 디렉터리

여러 features와 entities를 조합하여 **재사용 가능한 UI 블록**을 구성하는 레이어입니다.
헤더, 사이드바, 카드 등 **둘 이상의 페이지에서 공유**되는 섹션을 여기서 구현합니다.

---

## 1. 역할 · 책임

| 구분             | 설명                                           |
| ---------------- | ---------------------------------------------- |
| **UI 조합**      | 여러 features/entities를 묶어 완결된 섹션 제공 |
| **재사용성**     | 다수 페이지에서 공통으로 사용 가능             |
| **프레젠테이션** | 시각적 배치 담당, 비즈니스 로직은 하위 레이어  |

---

## 2. 구조 · 파일 설명

```
widgets/
└─ [widget-name]/
   ├─ ui/                # UI 구현
   ├─ model/             # (선택) 위젯 전용 상태 관리
   ├─ lib/               # (선택) 위젯 전용 유틸리티
   └─ index.ts           # Public API
```

| 파일/폴더      | 역할 및 제약사항                                    |
| -------------- | --------------------------------------------------- |
| **폴더명**     | kebab-case 사용 (`app-header`, `side-nav` 등)       |
| **`ui/`**      | 위젯 UI 구현. 복잡도에 따라 파일 또는 폴더로 구성   |
| **`model/`**   | (선택) 위젯 전용 상태나 로직. 간단한 경우 생략 가능 |
| **`lib/`**     | (선택) 위젯 전용 유틸리티 함수                      |
| **`index.ts`** | Public API 정의. 외부에서는 이것만 import 허용      |

---

## 3. import 규칙

- **허용**: `features/*`, `entities/*`, `shared/*`
- **금지**: `pages/*`, `app/*`, `widgets/*` (다른 위젯)

---

## 4. 위젯 생성 기준

### 언제 위젯으로 만들까?

- **2개 이상의 페이지**에서 동일한 UI 블록이 필요한 경우
- 예: 헤더, 사이드바, 공통 카드 컴포넌트

### 언제 페이지 내부에 구현할까?

- **특정 페이지에서만** 사용하는 UI 블록
- 해당 페이지의 `ui/` 폴더 내부에 컴포넌트로 구현
