# `widgets/` 디렉터리

`widgets` 레이어는 **“여러 features · entities를 조합해 하나의 화면 구역(section)을 이루는 재사용 가능한 UI 블록”** 을 담습니다.
헤더, 사이드바, 카드형 패널처럼 **둘 이상의 페이지에서 공유**되거나 동일 페이지 내에서 반복 배치되는 섹션을 여기서 구현하세요.

> **레벨 관계** `pages` → **widgets** → features → entities → shared  
> • `widgets`는 상위 레이어(pages·app)를 import할 수 없습니다.  
> • 하위 레이어(features·entities·shared)만 사용할 수 있습니다.

---

## 1. 역할 · 책임

| 항목                  | 설명                                                                                     |
| --------------------- | ---------------------------------------------------------------------------------------- |
| **조립(Composition)** | 둘 이상의 feature / entity를 묶어 “완결된 섹션”을 제공                                   |
| **프레젠테이션**      | 시각적·배치 책임을 지며, 비즈니스 로직은 feature 내부에 한정                             |
| **재사용성**          | 여러 페이지 또는 동일 페이지 내 다른 위치에서 그대로 사용 가능                           |
| **Public API**        | `index.ts`로만 공개 → 외부에서는 `import { Header } from "@/widgets/header"` 식으로 사용 |

---

## 2. 구조 & 명명 규칙

```
widgets/
└─ counter-panel/                ← kebab-case
   ├─ ui.tsx                     ← 대표 컴포넌트 (default export)
   ├─ index.ts                   ← export { default as CounterPanel } from './ui'
   ├─ counter-panel.stories.tsx  ← (선택) Storybook
   ├─ counter-panel.test.tsx     ← (선택) 테스트
   └─ README.md                  ← (선택) 위젯 전용 문서
```

| 규칙              | 내용                                                      |
| ----------------- | --------------------------------------------------------- |
| **폴더**          | 항상 **kebab-case** (`header`, `side-nav`, `score-board`) |
| **핵심 파일**     | `ui.tsx` + `index.ts`                                     |
| **추가 파일**     | 테스트, 스토리북, 스타일 모듈 등 필요 시 생성             |
| **컴포넌트 이름** | 코드 내부에서는 **PascalCase** (`Header`, `CounterPanel`) |

---

## 3. 생성 절차

1. **폴더 이름**  `src/widgets/<widget-name>/` — kebab-case.
2. **필수 파일** `ui.tsx` (구현)   `index.ts` (배럴).
3. **Public API**   외부에서 가져올 컴포넌트·함수를 전부 `index.ts`에 export.
4. (선택) 테스트 / 스토리북 파일 추가.

---

## 4. import 허용 · 금지

| 허용(✓)                                                                      | 금지(✗)              |
| ---------------------------------------------------------------------------- | -------------------- |
| `widgets/*` (자체 위젯 재사용)<br>`features/*`<br>`entities/*`<br>`shared/*` | `pages/*`<br>`app/*` |

ESLint 규칙 `eslint-plugin-fsd-lint/layer-imports`가 위반 시 오류를 표시합니다.

## 5. FAQ

| 질문                                              | 답변                                                                                                                          |
| ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **위젯 안에서 또 다른 위젯을 import해도 되나요?** | 가능하지만 의존성이 깊어지지 않도록 주의하세요. 보통은 `shared`의 프리미티브 UI나 `features`를 직접 조합하는 편이 깔끔합니다. |
| **스타일을 전용 CSS 모듈로 분리해도 되나요?**     | 가능합니다. 다만 Tailwind 클래스로 표현할 수 있는 경우 우선 Tailwind를 사용해 통일된 디자인 토큰을 유지하세요.                |
| **페이지 전용 레이아웃과 위젯의 차이는?**         | 레이아웃은 페이지 폴더에서만 쓰이며 URL마다 달라질 수 있습니다. 위젯은 여러 페이지에 재사용될 수 있는 독립 섹션입니다.        |
